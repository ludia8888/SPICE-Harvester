# 액션 템플릿(Action Templates)

액션 템플릿(Action Templates)을 활용하면 코드 없이도 재사용 가능한 비즈니스 로직을 정의할 수 있어요. 이 가이드에서는 비즈니스 분석가를 위해 액션 템플릿의 생성과 활용법을 알아봅니다.

## 액션이란?

**액션(Actions)**은 Spice OS 플랫폼의 쓰기(Write) 측면을 담당합니다. 데이터를 직접 변경하는 대신, 모든 변경 사항은 액션 시스템을 통해 처리되며, 다음과 같은 기능을 제공합니다:

- **유효성 검사(Validation)** -- 액션은 실행 전에 규칙에 따라 검증됩니다
- **감사 추적(Audit Trail)** -- 모든 액션은 불변 이벤트로 기록됩니다
- **실행 취소 지원(Undo Support)** -- 보상 액션(Compensating Actions)을 통해 액션을 되돌릴 수 있습니다
- **충돌 해결(Conflict Resolution)** -- 동시 수정이 감지되고 해결됩니다

### 액션 유형

| Action Type | 설명 | 예시 |
|------------|------|------|
| `createObject` | 새 인스턴스 생성 | 새 직원 추가 |
| `editObject` | 속성 수정 | 부서 변경 |
| `deleteObject` | 인스턴스 삭제 | 직원 비활성화 |
| `createLink` | 관계 생성 | 직원을 프로젝트에 배정 |
| `deleteLink` | 관계 삭제 | 직원 배정 해제 |
| `bulkAction` | 여러 액션을 원자적으로 실행 | 일괄 온보딩 |

## 기본 제공 액션 사용하기

### 객체 생성

```bash
POST /api/v2/ontologies/acme/actions/createObject

{
  "parameters": {
    "objectType": "Employee",
    "properties": {
      "employeeId": "EMP-099",
      "fullName": "John Smith",
      "department": "Marketing",
      "startDate": "2024-01-15",
      "status": "ACTIVE"
    }
  }
}
```

### 객체 수정

```bash
POST /api/v2/ontologies/acme/actions/editObject

{
  "parameters": {
    "objectType": "Employee",
    "primaryKey": "EMP-099",
    "properties": {
      "department": "Sales",
      "status": "ACTIVE"
    }
  }
}
```

### 객체 삭제

```bash
POST /api/v2/ontologies/acme/actions/deleteObject

{
  "parameters": {
    "objectType": "Employee",
    "primaryKey": "EMP-099"
  }
}
```

### 적용 전 시뮬레이션

변경 사항 없이 액션을 테스트합니다:

```bash
POST /api/v2/ontologies/acme/actions/editObject/simulate

{
  "parameters": {
    "objectType": "Employee",
    "primaryKey": "EMP-042",
    "properties": { "department": "Research" }
  }
}
```

## 액션 템플릿

**액션 템플릿(Action Template)**은 동적 표현식(Dynamic Expressions)을 포함하는 재사용 가능한 액션 정의입니다. 템플릿은 실행 시점에 평가되며, 입력값, 현재 사용자, 타임스탬프, 기존 객체 속성을 참조할 수 있습니다.

### 템플릿 버전

| 버전 | 설명 | 기능 |
|------|------|------|
| `template_v1` | 기본 템플릿 | `$ref`, `$now`만 지원 |
| `template_v2` | 고급 템플릿 | `$ref`, `$now`, `$if`, `$switch`, `$call` 및 전체 표현식 지원 |
| `function_v1` | 함수 기반 | 사용자 정의 함수 구현 |

## 템플릿 표현식(Template Expressions)

### `$ref` -- 값 참조

입력 매개변수, 사용자 컨텍스트 또는 대상 객체 속성을 참조합니다:

```json
{
  "$ref": "input.department"
}
```

| 참조 대상 | 설명 | 예시 |
|-----------|------|------|
| `input.<field>` | 입력 매개변수 값 | `{"$ref": "input.newStatus"}` |
| `user.id` | 현재 사용자의 ID | `{"$ref": "user.id"}` |
| `user.name` | 현재 사용자의 이름 | `{"$ref": "user.name"}` |
| `target.<field>` | 기존 객체 속성 | `{"$ref": "target.currentStatus"}` |

### `$now` -- 현재 타임스탬프

현재 ISO 8601 날짜/시간을 삽입합니다:

```json
{
  "$now": true
}
```

결과: `"2024-01-15T09:30:00Z"`

### `$if` -- 조건부 로직

조건에 따라 분기합니다:

```json
{
  "$if": {
    "cond": { "$gt": [{ "$ref": "input.amount" }, 10000] },
    "then": "REQUIRES_APPROVAL",
    "else": "AUTO_APPROVED"
  }
}
```

### `$switch` -- 다중 분기 조건

여러 조건을 순차적으로 평가합니다:

```json
{
  "$switch": {
    "cases": [
      {
        "when": { "$eq": [{ "$ref": "input.priority" }, "critical"] },
        "then": "ESCALATED"
      },
      {
        "when": { "$eq": [{ "$ref": "input.priority" }, "high"] },
        "then": "URGENT"
      }
    ],
    "default": "NORMAL"
  }
}
```

### `$call` -- 함수 호출

내장 함수를 호출합니다:

```json
{
  "$call": {
    "fn": "upper",
    "args": [{ "$ref": "input.code" }]
  }
}
```

**사용 가능한 함수:**

| 함수 | 설명 | 예시 |
|------|------|------|
| `upper` | 대문자 변환 | `upper("hello")` → `"HELLO"` |
| `lower` | 소문자 변환 | `lower("HELLO")` → `"hello"` |
| `concat` | 문자열 연결 | `concat("A", "-", "B")` → `"A-B"` |
| `add` | 숫자 덧셈 | `add(10, 5)` → `15` |
| `multiply` | 숫자 곱셈 | `multiply(10, 1.1)` → `11` |
| `divide` | 숫자 나눗셈 | `divide(100, 4)` → `25` |

### 비교 연산자(조건문에서 사용)

| 연산자 | 설명 |
|--------|------|
| `$eq` | 같음 |
| `$gt` | 보다 큼 |
| `$gte` | 크거나 같음 |
| `$lt` | 보다 작음 |
| `$lte` | 작거나 같음 |

## 템플릿 구조

완전한 액션 템플릿 정의:

```json
{
  "type": "template_v2",
  "targets": [
    {
      "target": { "from": "input.ticket" },
      "changes": {
        "set": {
          "status": {
            "$if": {
              "cond": { "$gt": [{ "$ref": "input.amount" }, 10000] },
              "then": "REQUIRES_APPROVAL",
              "else": "AUTO_APPROVED"
            }
          },
          "reviewer": {
            "$call": {
              "fn": "upper",
              "args": [{ "$ref": "user.id" }]
            }
          },
          "reviewedAt": { "$now": true }
        },
        "unset": ["rejected_reason"],
        "link_add": [
          { "field": "assignees", "value": { "$ref": "input.assigneeId" } }
        ],
        "delete": false
      }
    }
  ]
}
```

### 변경 작업(Change Operations)

| 작업 | 설명 | 예시 |
|------|------|------|
| `set` | 속성 값 설정 | `{"set": {"status": "ACTIVE"}}` |
| `unset` | 속성 제거 | `{"unset": ["tempField"]}` |
| `link_add` | 링크 관계 추가 | `{"link_add": [{"field": "tags", "value": "urgent"}]}` |
| `delete` | 대상 객체 삭제 | `{"delete": true}` |

## 일반적인 패턴

### 직원 온보딩(Employee Onboarding)

기본값을 자동으로 설정하고 부서에 연결합니다:

```json
{
  "type": "template_v2",
  "targets": [
    {
      "target": { "from": "input.employee" },
      "changes": {
        "set": {
          "status": "ACTIVE",
          "startDate": { "$now": true },
          "onboardedBy": { "$ref": "user.id" },
          "accessLevel": {
            "$switch": {
              "cases": [
                {
                  "when": { "$eq": [{ "$ref": "input.role" }, "manager"] },
                  "then": "FULL"
                },
                {
                  "when": { "$eq": [{ "$ref": "input.role" }, "contractor"] },
                  "then": "LIMITED"
                }
              ],
              "default": "STANDARD"
            }
          }
        },
        "link_add": [
          { "field": "department", "value": { "$ref": "input.departmentId" } }
        ]
      }
    }
  ]
}
```

### 승인 워크플로우(Approval Workflow)

금액 임계값에 따라 항목을 라우팅합니다:

```json
{
  "type": "template_v2",
  "targets": [
    {
      "target": { "from": "input.request" },
      "changes": {
        "set": {
          "approvalStatus": {
            "$if": {
              "cond": { "$gt": [{ "$ref": "target.amount" }, 50000] },
              "then": "EXECUTIVE_REVIEW",
              "else": {
                "$if": {
                  "cond": { "$gt": [{ "$ref": "target.amount" }, 10000] },
                  "then": "MANAGER_REVIEW",
                  "else": "AUTO_APPROVED"
                }
              }
            }
          },
          "lastReviewedAt": { "$now": true },
          "lastReviewedBy": { "$ref": "user.id" }
        }
      }
    }
  ]
}
```

### 상태 전환(Status Transition)

유효한 상태 전환을 적용합니다:

```json
{
  "type": "template_v2",
  "targets": [
    {
      "target": { "from": "input.ticket" },
      "changes": {
        "set": {
          "status": { "$ref": "input.newStatus" },
          "statusChangedAt": { "$now": true },
          "statusChangedBy": { "$ref": "user.id" },
          "previousStatus": { "$ref": "target.status" }
        }
      }
    }
  ]
}
```

## 일괄 액션(Batch Actions)

여러 액션을 원자적으로 실행합니다:

```bash
POST /api/v2/ontologies/acme/actions/createObject/batch

{
  "requests": [
    {
      "parameters": {
        "objectType": "Employee",
        "properties": { "employeeId": "EMP-100", "fullName": "Alice" }
      }
    },
    {
      "parameters": {
        "objectType": "Employee",
        "properties": { "employeeId": "EMP-101", "fullName": "Bob" }
      }
    }
  ]
}
```

## 템플릿 테스트

적용하기 전에 항상 시뮬레이션을 수행합니다:

1. **시뮬레이션(Simulate)** -- 부작용 없이 액션을 테스트합니다
2. **검토(Review)** -- 계산된 값이 올바른지 확인합니다
3. **적용(Apply)** -- 실제로 액션을 실행합니다

시뮬레이션 엔드포인트(Simulate Endpoint)는 변경 사항을 적용하지 않고 어떤 일이 발생할지를 반환하므로, 템플릿 표현식이 예상한 결과를 생성하는지 확인할 수 있습니다.
