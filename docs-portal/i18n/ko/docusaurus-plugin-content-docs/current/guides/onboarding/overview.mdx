# 온보딩 개요

Spice OS에 오신 것을 환영합니다!

이 가이드는 처음 합류한 팀원이 빠르게 적응할 수 있도록 역할별 학습 경로를 안내합니다.
데이터 엔지니어, 플랫폼 관리자, 통합 개발자, 비즈니스 분석가 등 어떤 역할이든 상관없어요. 이 페이지를 출발점으로 삼으면 됩니다.

## Spice OS란?

Spice OS는 Palantir Foundry에서 영감을 받은 **엔터프라이즈 온톨로지 및 데이터 플랫폼**이에요.
한마디로, 조직의 데이터를 하나의 모델로 통합하고 안전하게 관리할 수 있는 시스템이라고 생각하면 됩니다.

주요 기능을 간단히 살펴볼까요?

- **온톨로지 기반 데이터 모델** -- 강타입 스키마로 객체 유형(Object Type)/속성(Property)/관계를 정의해요
- **데이터 수집(Ingestion)** -- PostgreSQL, MySQL, Oracle, Snowflake, SQL Server, Google Sheets 등 6개 외부 소스를 5가지 동기화 모드로 연결할 수 있어요
- **파이프라인(Pipeline) 빌더** -- 22개 변환 타입과 UDF, 3가지 실행 모드(Preview/Build/Deploy)로 데이터를 변환해요
- **Foundry v2 API** -- Palantir Foundry API와 호환되는 234개 이상의 REST 엔드포인트를 제공합니다
- **이벤트 소싱** -- 모든 변경을 불변 이벤트로 기록하기 때문에 완전한 감사 추적이 가능해요
- **버전 관리 데이터셋** -- LakeFS를 활용해 Git처럼 데이터셋을 브랜칭하고 버저닝할 수 있어요

## 역할별 학습 경로

자신의 역할에 맞는 경로를 골라 순서대로 따라가 보세요.

### 데이터 엔지니어

데이터 모델과 파이프라인을 설계하고 운영하는 역할이에요.

| 단계 | 가이드 | 학습 내용 |
|------|-------|-------------------|
| 1 | [핵심 개념](/docs/getting-started/concepts) | 온톨로지, 객체 유형, 속성, 링크 유형(Link Type), 데이터셋 |
| 2 | [스키마 구성](../data-engineer/schema-config) | 객체 유형 정의, 속성 구성, 링크 유형 생성 |
| 3 | [임포트 템플릿](../data-engineer/import-templates) | 외부 데이터 소스 연결, 동기화 모드 구성 |
| 4 | [파이프라인 빌더](../data-engineer/pipeline-builder) | 변환 파이프라인 구성, UDF 사용, 실행 스케줄링 |

**첫 번째 과제:** 속성 3개를 가진 객체 유형을 정의하고, CSV 파일에서 데이터를 로드해 보세요.

### 플랫폼 관리자

플랫폼 배포, 모니터링, 보안 설정을 담당합니다.

| 단계 | 가이드 | 학습 내용 |
|------|-------|-------------------|
| 1 | [설치](/docs/getting-started/installation) | Docker Compose로 26개 서비스 배포 |
| 2 | [아키텍처 투어](./architecture-tour) | 서비스 간 상호작용과 데이터 흐름 파악 |
| 3 | [서비스 모니터링](../platform-admin/service-monitoring) | 헬스 체크, Docker 모니터링, 문제 해결 |
| 4 | [RBAC](../platform-admin/rbac) | 사용자, 역할, 권한, 멀티 테넌시 |

**첫 번째 과제:** 로컬에 플랫폼을 배포한 뒤, 모든 헬스 엔드포인트가 `healthy`를 반환하는지 확인해 보세요.

### 통합 개발자

Spice OS API를 활용하는 애플리케이션을 만드는 역할이에요.

| 단계 | 가이드 | 학습 내용 |
|------|-------|-------------------|
| 1 | [핵심 개념](/docs/getting-started/concepts) | 데이터 모델과 API 개념 |
| 2 | [REST API](../integration-developer/rest-api) | 엔드포인트, 검색, 페이지네이션, 에러 처리 |
| 3 | [인증 플로우](../integration-developer/auth-flow) | 인증 전략, JWT, 토큰 |
| 4 | [SDK 가이드](../integration-developer/sdk-guide) | Python/TypeScript 클라이언트 라이브러리 |

**첫 번째 과제:** Bearer 토큰으로 인증한 뒤, 온톨로지에 등록된 모든 객체 유형을 조회해 보세요.

### 비즈니스 분석가

데이터를 검색/분석하고, 액션(Action)을 실행하는 역할입니다.

| 단계 | 가이드 | 학습 내용 |
|------|-------|-------------------|
| 1 | [핵심 개념](/docs/getting-started/concepts) | 객체 유형, 인스턴스(Instance), 액션 |
| 2 | [검색 & 쿼리](../business-analyst/search-query) | 23개 검색 연산자, 필터, 정렬 |
| 3 | [액션 템플릿](../business-analyst/action-templates) | 표현식을 사용한 재사용 가능한 액션 로직 |
| 4 | [대시보드](../business-analyst/dashboard) | 데이터 시각화 및 탐색 |

**첫 번째 과제:** `eq` 연산자로 객체를 검색하고, 결과를 이름순으로 정렬해 보세요.

## 핵심 용어

처음 접하면 헷갈릴 수 있는 용어들을 정리했어요. 필요할 때마다 돌아와서 확인하세요.

| 용어 | 정의 |
|------|-----------|
| **Ontology** | 데이터 모델 전체를 담는 최상위 컨테이너 |
| **Object Type** | 스키마 청사진이에요. 데이터베이스 테이블과 비슷한 개념입니다 |
| **Property** | 객체 유형에 속하는 타입이 지정된 속성 |
| **Instance** | 객체 유형의 실제 데이터 레코드 한 건 |
| **Link Type** | 객체 유형 사이의 방향성 있는 관계 |
| **Action** | 검증과 감사가 수반되는 쓰기 작업 |
| **Dataset** | LakeFS에 저장되는 버전 관리 데이터 단위 |
| **Pipeline** | 데이터 처리를 위한 DAG 기반 변환 흐름 |
| **Connection** | 외부 데이터 소스와의 연결 설정 |
| **Projection** | 조회 성능을 위해 미리 계산해 둔 뷰 |
| **RID** | 리소스 식별자 (`ri.spice.main.{type}.{id}` 형식) |
| **Event** | 상태 변경을 기록한 불변 이벤트 |

## 대표 워크플로

각 작업이 어떤 순서로 진행되는지 한눈에 파악할 수 있어요.

### 데이터 수집

```
Create Connection → Configure Import → Choose Sync Mode → Execute → Verify Data
```

### 스키마 설계

```
Define Object Types → Add Properties → Create Link Types → Load Sample Data → Iterate
```

### 통합 개발

```
Get Token → Explore API → Build Queries → Handle Pagination → Implement Error Handling
```

### 데이터 변환

```
Create Pipeline → Add Transforms → Preview → Build → Deploy → Schedule
```

## 환경 접근하기

### 로컬 개발

```bash
# 저장소 클론
git clone <repository-url>
cd SPICE-Harvester/backend

# 전체 서비스 시작
docker compose up -d

# 헬스 확인
curl http://localhost:8080/api/health
```

### API 접근

| 서비스 | URL | 용도 |
|---------|-----|---------|
| BFF API | `http://localhost:8080/api` | 기본 API 게이트웨이 |
| OMS API | `http://localhost:8000` | 온톨로지 관리 |
| MinIO Console | `http://localhost:9001` | 오브젝트 스토리지 UI |
| lakeFS | `http://localhost:48080` | 데이터 버저닝 UI |
| Elasticsearch | `http://localhost:9200` | 검색 엔진 |

### 자격 증명 얻기

1. `.env` 또는 `docker-compose.yml`에서 기본 토큰을 확인하세요
2. JWT 인증을 사용한다면 IdP를 구성해야 해요 ([인증 플로우](../integration-developer/auth-flow) 참고)
3. 개발 환경에서는 Dev Master Auth를 활성화할 수도 있어요 (개발 전용)
