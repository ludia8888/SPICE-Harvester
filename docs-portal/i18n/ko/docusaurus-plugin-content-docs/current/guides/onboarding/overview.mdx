# 온보딩 개요

Spice OS에 오신 것을 환영합니다. 이 가이드는 신규 팀원을 위한 구조화된 온보딩 경로를 제공합니다. 데이터 엔지니어, 플랫폼 관리자, 통합 개발자, 비즈니스 분석가 등 어떤 역할이든, 이 페이지를 통해 빠르게 시작할 수 있습니다.

## Spice OS란?

Spice OS는 Palantir Foundry에서 영감을 받은 **엔터프라이즈 온톨로지 및 데이터 플랫폼**입니다. 다음을 제공합니다:

- **온톨로지 기반 데이터 모델** -- 강타입 스키마로 객체 유형/속성/관계를 정의
- **데이터 수집(Ingestion)** -- 6개 외부 데이터 소스(PostgreSQL/MySQL/Oracle/Snowflake/SQL Server/Google Sheets)에 대해 5개 동기화 모드 지원
- **파이프라인 빌더** -- 22개 변환 타입, UDF, 3개 실행 모드(Preview/Build/Deploy)로 데이터 변환
- **Foundry v2 API** -- Palantir Foundry API 표면과 호환되는 234개 이상의 REST 엔드포인트
- **이벤트 소싱** -- 모든 변이를 불변 이벤트로 캡처(완전한 감사 추적)
- **버전 관리 데이터셋** -- LakeFS를 통한 Git 유사 브랜칭/버저닝

### 데이터 엔지니어

| 단계 | 가이드 | 학습 내용 |
|------|-------|-------------------|
| 1 | [핵심 개념](/docs/getting-started/concepts) | 온톨로지, 객체 유형, 속성, 링크 유형, 데이터셋 |
| 2 | [스키마 구성](../data-engineer/schema-config) | 객체 유형 정의, 속성 구성, 링크 유형 생성 |
| 3 | [임포트 템플릿](../data-engineer/import-templates) | 외부 데이터 소스 연결, 동기화 모드 구성 |

## 핵심 용어

| 용어 | 정의 |
|------|-----------|
| **Ontology** | 데이터 모델을 정의하는 최상위 컨테이너 |
| **Object Type** | 스키마 청사진(데이터베이스 테이블과 유사) |
| **Property** | 객체 유형의 타입이 있는 속성 |
| **Instance** | 객체 유형의 실제 레코드 |
| **Link Type** | 객체 유형 간 방향성 있는 관계 |
| **Action** | 검증되고 감사되는 쓰기 작업 |
| **Dataset** | LakeFS에 저장되는 버전 관리 데이터 객체 |
| **Pipeline** | 데이터 처리를 위한 DAG 변환 노드 |
| **Connection** | 외부 데이터 소스에 대한 구성된 연결 |
| **Projection** | 쿼리에 최적화된 사전 계산 뷰 |
| **RID** | Resource Identifier (`ri.spice.main.{type}.{id}`) |
| **Event** | 상태 변경의 불변 기록 |

## 환경 접근

### 로컬 개발

```bash
# 저장소 클론
git clone <repository-url>
cd SPICE-Harvester/backend

# 전체 서비스 시작
docker compose up -d

# 헬스 확인
curl http://localhost:8080/api/health
```

### API 접근

| 서비스 | URL | 용도 |
|---------|-----|---------|
| BFF API | `http://localhost:8080/api` | 기본 API 게이트웨이 |
| OMS API | `http://localhost:8000` | 온톨로지 관리 |
| MinIO Console | `http://localhost:9001` | 오브젝트 스토리지 UI |
| lakeFS | `http://localhost:48080` | 데이터 버저닝 UI |
| Elasticsearch | `http://localhost:9200` | 검색 엔진 |

### 자격 증명 얻기

1. `.env` 또는 `docker-compose.yml`에서 기본 토큰 확인
2. JWT 인증을 사용한다면 IdP를 구성합니다([인증 플로우](../integration-developer/auth-flow) 참고)
3. 개발 환경에서는 Dev Master Auth를 활성화할 수 있습니다(개발 전용)
