# 빠른 시작

10분 안에 로컬에서 Spice OS를 실행해봅시다. 사전 준비부터 플랫폼 실행, 첫 API 호출까지 단계별로 안내합니다.

## 사전 준비

시작하기 전에 개발 머신에 아래 도구가 설치되어 있는지 확인하세요.

| 도구 | 최소 버전 | 확인 명령 |
|------|----------|----------|
| Docker | 24.0+ | `docker --version` |
| Docker Compose | 2.20+ | `docker compose version` |
| Node.js | 20.0+ | `node --version` |
| Python | 3.11+ | `python3 --version` |
| Git | 2.40+ | `git --version` |

**시스템 요구 사항**

- RAM 최소 16 GB (관측성 스택까지 사용하려면 32 GB 권장)
- 디스크 여유 공간 20 GB
- macOS 13+, Ubuntu 22.04+, 또는 WSL2가 포함된 Windows 11

## 1단계: 저장소 클론

```bash
git clone https://github.com/your-org/spice-harvester.git
cd spice-harvester
```

## 2단계: 환경 변수 설정

예제 환경 파일을 복사한 뒤, 필요에 따라 값을 조정하세요.

```bash
cp .env.example .env
```

기본 설정은 로컬 개발에서 바로 동작하도록 구성되어 있어요. 주요 기본값은 다음과 같습니다.

```bash
# Core service ports
OMS_PORT=8000           # Ontology Management Service
BFF_PORT=8002           # Backend-for-Frontend gateway
AGENT_PORT=8004         # Agent tool runner service

# Data stores
POSTGRES_PORT=5432
ELASTICSEARCH_PORT=9200
REDIS_PORT=6379
KAFKA_PORT=9092
MINIO_PORT=9000
```

## 3단계: 플랫폼 실행

Docker Compose로 32개 서비스를 모두 실행합니다.

```bash
docker compose up -d
```

이 명령으로 실행되는 전체 스택은 다음과 같아요.

- **API 서버 3개** -- OMS, BFF, Agent
- **워커 12개** -- 액션 워커, 프로젝션 빌더, 파이프라인 러너, 커넥터 동기화 등
- **데이터 플랫폼 서비스 7개** -- PostgreSQL, Elasticsearch, Redis, Kafka, ZooKeeper, MinIO, LakeFS
- **관측성 서비스 6개** -- Grafana, Prometheus, Jaeger, OTEL Collector, AlertManager, Kafka UI

:::tip
개발 중 가볍게 실행하려면 핵심 서비스만 시작하세요:
```bash
docker compose --profile core up -d
```
:::

## 4단계: 서비스 확인

모든 서비스가 정상인지 확인해봅시다.

```bash
docker compose ps
```

또는 헬스 엔드포인트를 직접 호출할 수도 있습니다.

```bash
# OMS health check
curl http://localhost:8000/health

# BFF health check
curl http://localhost:8002/health

# Agent health check
curl http://localhost:8004/health
```

각 엔드포인트는 다음과 같은 응답을 반환해야 합니다.

```json
{
  "status": "healthy",
  "version": "1.0.0",
  "uptime": "...",
  "checks": {
    "database": "ok",
    "elasticsearch": "ok",
    "redis": "ok",
    "kafka": "ok"
  }
}
```

## 5단계: 데이터베이스 마이그레이션 실행

초기 스키마와 시드 데이터를 적용합니다.

```bash
docker compose exec oms python -m alembic upgrade head
docker compose exec oms python -m spice.cli seed --demo
```

`seed` 명령은 샘플 객체 유형, 링크 유형, 인스턴스를 포함한 데모 온톨로지를 생성합니다. 이 데이터로 바로 플랫폼을 탐색할 수 있어요.

## 6단계: 첫 API 호출

### 인증

먼저 Bearer 토큰을 발급받습니다.

```bash
curl -X POST http://localhost:8002/api/v2/auth/token \
  -H "Content-Type: application/json" \
  -d '{
    "username": "admin",
    "password": "admin"
  }'
```

정상적으로 처리되면 다음과 같은 응답을 받게 됩니다.

```json
{
  "access_token": "eyJhbGciOiJIUzI1NiIs...",
  "token_type": "Bearer",
  "expires_in": 3600
}
```

### 객체 유형 목록 조회

발급받은 토큰으로 데모 온톨로지의 객체 유형 목록을 조회해볼까요.

```bash
curl http://localhost:8002/api/v2/ontologies/demo/objectTypes \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIs..."
```

다음과 같은 응답을 확인할 수 있습니다.

```json
{
  "data": [
    {
      "apiName": "Employee",
      "displayName": "Employee",
      "pluralDisplayName": "Employees",
      "primaryKey": "employeeId",
      "properties": {
        "employeeId": { "dataType": { "type": "string" } },
        "fullName": { "dataType": { "type": "string" } },
        "department": { "dataType": { "type": "string" } },
        "startDate": { "dataType": { "type": "date" } }
      },
      "rid": "ri.ontology.main.object-type.employee-001"
    }
  ],
  "nextPageToken": null
}
```

### 객체 검색

Engineering 부서의 직원을 검색해봅시다.

```bash
curl -X POST http://localhost:8002/api/v2/ontologies/demo/objects/Employee/search \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIs..." \
  -H "Content-Type: application/json" \
  -d '{
    "where": {
      "type": "eq",
      "field": "department",
      "value": "Engineering"
    },
    "pageSize": 10
  }'
```
