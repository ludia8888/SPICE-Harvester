# Foundry v2 검색 API

검색 API는 `SearchJsonQueryV2` 형식으로 객체를 쿼리해요. **13개 연산자**를 조합해 다양한 필터를 만들 수 있고, Elasticsearch 기반으로 Foundry v2 표준 형식의 결과를 돌려줍니다.

## 객체 검색

```
POST /api/v2/ontologies/{ontologyApiName}/objects/{objectTypeApiName}/search
```

### 경로 파라미터

| 파라미터 | 타입 | 필수 | 설명 |
|:---------|:-----|:-----|:-----|
| `ontologyApiName` | string | 예 | 온톨로지의 API 이름 |
| `objectTypeApiName` | string | 예 | 검색할 객체 유형의 API 이름 |

### 요청 바디

```json
{
  "where": { ... },
  "orderBy": [
    { "field": "startDate", "direction": "desc" }
  ],
  "select": ["fullName", "department", "startDate"],
  "pageSize": 25,
  "pageToken": null
}
```

| 필드 | 타입 | 필수 | 설명 |
|:-----|:-----|:-----|:-----|
| `where` | SearchJsonQueryV2 | 예 | 검색 연산자를 사용한 필터 표현식 |
| `orderBy` | array | 아니오 | 정렬 순서. `field`와 `direction` (`asc` 또는 `desc`) |
| `select` | string[] | 아니오 | 결과에 포함할 속성 |
| `pageSize` | integer | 아니오 | 페이지당 결과 수 (기본값: 100, 최대: 1000) |
| `pageToken` | string | 아니오 | 다음 페이지용 토큰 |

## SearchJsonQueryV2 연산자

`where` 절은 재귀적 JSON 구조를 사용해요. 각 연산자는 `type` 필드로 구분합니다.

### 비교 연산자

#### `eq` -- 같음

```json
{ "type": "eq", "field": "department", "value": "Engineering" }
```

#### `gt` / `gte` -- 초과 / 이상

```json
{ "type": "gt", "field": "salary", "value": 100000 }
{ "type": "gte", "field": "startDate", "value": "2023-01-01" }
```

#### `lt` / `lte` -- 미만 / 이하

```json
{ "type": "lt", "field": "salary", "value": 200000 }
{ "type": "lte", "field": "startDate", "value": "2024-12-31" }
```

#### `isNull` -- NULL 체크

```json
{ "type": "isNull", "field": "department", "value": true }
```

`value`를 `false`로 설정하면 NULL이 아닌 객체만 찾아요.

---

### 텍스트 검색 연산자

#### `containsAnyTerm` -- 임의 텀 매칭

텍스트 필드에 지정된 텀 중 **하나라도** 포함된 객체를 찾아요.

```json
{ "type": "containsAnyTerm", "field": "fullName", "value": "jane john" }
```

#### `containsAllTerms` -- 모든 텀 매칭

텍스트 필드에 지정된 텀이 **모두** 들어 있는 객체를 찾아요. 순서는 상관없어요.

```json
{ "type": "containsAllTerms", "field": "description", "value": "data pipeline" }
```

#### `containsAllTermsInOrder` -- 구문 매칭 (Phrase Match)

지정된 텀이 **정확한 순서**로 포함된 객체를 찾아요.

```json
{ "type": "containsAllTermsInOrder", "field": "description", "value": "data transformation pipeline" }
```

#### `containsAllTermsInOrderPrefixLastTerm` -- 접두사 구문 매칭

마지막 텀을 접두사(prefix)로 처리해요. **자동완성이나 타이프어헤드** 구현에 유용합니다.

```json
{ "type": "containsAllTermsInOrderPrefixLastTerm", "field": "fullName", "value": "jane d" }
```

→ "Jane Doe", "Jane Davis" 등을 매칭

#### `startsWith` -- 접두사 매칭

키워드(비분석) 필드에서 접두사로 시작하는 값을 찾아요.

```json
{ "type": "startsWith", "field": "employeeId", "value": "EMP-04" }
```

---

### 논리 연산자

#### `and` -- 논리 AND

여러 조건을 결합해요. **모든 조건**이 매칭되어야 합니다.

```json
{
  "type": "and",
  "value": [
    { "type": "eq", "field": "department", "value": "Engineering" },
    { "type": "gte", "field": "salary", "value": 120000 },
    { "type": "eq", "field": "isActive", "value": true }
  ]
}
```

#### `or` -- 논리 OR

여러 조건 중 **하나 이상**만 매칭되면 돼요.

```json
{
  "type": "or",
  "value": [
    { "type": "eq", "field": "department", "value": "Engineering" },
    { "type": "eq", "field": "department", "value": "Product" }
  ]
}
```

#### `not` -- 논리 NOT

조건을 반전시켜요.

```json
{
  "type": "not",
  "value": { "type": "eq", "field": "department", "value": "HR" }
}
```

## 전체 요청/응답 예시

### 요청

2023년 이후 입사, 연봉 10만 이상인 현직 엔지니어를 입사일 내림차순으로 검색하는 예시를 볼까요.

```bash
curl -X POST \
  "http://localhost:8002/api/v2/ontologies/acme/objects/Employee/search" \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "where": {
      "type": "and",
      "value": [
        { "type": "eq", "field": "department", "value": "Engineering" },
        { "type": "gte", "field": "startDate", "value": "2023-01-01" },
        { "type": "gt", "field": "salary", "value": 100000 },
        { "type": "eq", "field": "isActive", "value": true }
      ]
    },
    "orderBy": [{ "field": "startDate", "direction": "desc" }],
    "select": ["fullName", "department", "startDate", "salary"],
    "pageSize": 10
  }'
```

### 응답

```json
{
  "data": [
    {
      "__rid": "ri.ontology.main.object.emp-100",
      "__primaryKey": "EMP-100",
      "__apiName": "Employee",
      "properties": {
        "fullName": "Alex Rivera",
        "department": "Engineering",
        "startDate": "2024-01-10",
        "salary": 155000.00
      }
    }
  ],
  "nextPageToken": null,
  "totalCount": 1
}
```

## 연산자 API 레퍼런스

| 연산자 | 분류 | 적용 가능 필드 | 설명 |
|:-------|:-----|:-------------|:-----|
| `eq` | 비교 | 모든 타입 | 정확 일치 |
| `gt` / `gte` | 비교 | 숫자, 날짜 | 초과 / 이상 |
| `lt` / `lte` | 비교 | 숫자, 날짜 | 미만 / 이하 |
| `isNull` | 비교 | 모든 타입 | NULL 체크 |
| `containsAnyTerm` | 텍스트 | 분석된 String | 임의 텀 매칭 |
| `containsAllTerms` | 텍스트 | 분석된 String | 모든 텀 매칭 (순서 무관) |
| `containsAllTermsInOrder` | 텍스트 | 분석된 String | 구문 매칭 |
| `containsAllTermsInOrderPrefixLastTerm` | 텍스트 | 분석된 String | 접두사 구문 매칭 |
| `startsWith` | 텍스트 | 키워드 String | 접두사 매칭 |
| `and` | 논리 | N/A | 모든 조건 충족 |
| `or` | 논리 | N/A | 하나 이상 충족 |
| `not` | 논리 | N/A | 조건 부정 |

:::note
`not`은 단일 쿼리를 감싸고, `and`와 `or`는 쿼리 배열을 감싸요. 논리 연산자는 원하는 만큼 깊이 중첩할 수 있습니다.
:::

## 에러 응답

| 상태 | 에러 코드 | 설명 |
|:-----|:---------|:-----|
| `400` | `InvalidQuery` | 검색 쿼리가 잘못되었거나 지원하지 않는 연산자 사용 |
| `400` | `InvalidField` | 지정한 필드가 객체 유형에 존재하지 않음 |
| `400` | `InvalidValue` | 값의 타입이 필드 타입과 불일치 |
| `404` | `ObjectTypeNotFound` | 객체 유형이 존재하지 않음 |
| `404` | `OntologyNotFound` | 온톨로지가 존재하지 않음 |
