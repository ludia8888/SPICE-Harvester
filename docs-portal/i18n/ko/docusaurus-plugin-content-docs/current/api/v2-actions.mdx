# Foundry v2 액션 API

액션(Action)은 Spice OS의 쓰기 인터페이스예요. 객체 생성, 속성(Property) 수정, 레코드 삭제, 링크 유형(Link Type) 관리 등 모든 변경 작업을 액션 API로 처리합니다. **검증, 감사 로깅, Undo, 충돌 해결**까지 기본 제공되거든요.

## 액션 실행

온톨로지에 하나 이상의 액션을 실행해요.

```
POST /api/v2/ontologies/{ontologyApiName}/actions/apply
```

### 경로 파라미터

| 파라미터 | 타입 | 필수 | 설명 |
|:---------|:-----|:-----|:-----|
| `ontologyApiName` | string | 예 | 온톨로지의 API 이름 |

### 요청 바디

```json
{
  "actions": [
    {
      "type": "createObject",
      "objectTypeApiName": "Employee",
      "properties": {
        "employeeId": "EMP-200",
        "fullName": "Morgan Chen",
        "department": "Engineering",
        "startDate": "2024-02-01",
        "isActive": true
      }
    }
  ]
}
```

---

### 액션 타입

#### `createObject` -- 객체 생성

```json
{
  "type": "createObject",
  "objectTypeApiName": "Employee",
  "properties": {
    "employeeId": "EMP-200",
    "fullName": "Morgan Chen",
    "department": "Engineering",
    "startDate": "2024-02-01"
  }
}
```

#### `editObject` -- 객체 수정

지정한 속성만 업데이트돼요. 나머지 속성은 그대로 유지됩니다.

```json
{
  "type": "editObject",
  "objectTypeApiName": "Employee",
  "primaryKey": "EMP-042",
  "properties": {
    "department": "Product",
    "salary": 160000.00
  }
}
```

#### `deleteObject` -- 객체 삭제

```json
{
  "type": "deleteObject",
  "objectTypeApiName": "Employee",
  "primaryKey": "EMP-042"
}
```

#### `createLink` -- 관계 생성

```json
{
  "type": "createLink",
  "linkTypeApiName": "manages",
  "sourceObjectPrimaryKey": "EMP-001",
  "sourceObjectType": "Employee",
  "targetObjectPrimaryKey": "PRJ-042",
  "targetObjectType": "Project"
}
```

#### `deleteLink` -- 관계 삭제

```json
{
  "type": "deleteLink",
  "linkTypeApiName": "manages",
  "sourceObjectPrimaryKey": "EMP-001",
  "sourceObjectType": "Employee",
  "targetObjectPrimaryKey": "PRJ-042",
  "targetObjectType": "Project"
}
```

---

### 벌크 액션

단일 요청에 여러 액션을 한꺼번에 보낼 수 있어요. **원자적(atomic)**으로 실행되기 때문에 모두 성공하거나 모두 실패합니다.

```bash
curl -X POST \
  "http://localhost:8002/api/v2/ontologies/acme/actions/apply" \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "actions": [
      {
        "type": "createObject",
        "objectTypeApiName": "Project",
        "properties": {
          "projectId": "PRJ-100",
          "name": "Atlas Initiative",
          "status": "active",
          "budget": 500000
        }
      },
      {
        "type": "createLink",
        "linkTypeApiName": "manages",
        "sourceObjectPrimaryKey": "EMP-042",
        "sourceObjectType": "Employee",
        "targetObjectPrimaryKey": "PRJ-100",
        "targetObjectType": "Project"
      }
    ]
  }'
```

### 응답

```json
{
  "actionResults": [
    {
      "actionId": "act-2024-01-15-a1b2c3",
      "type": "createObject",
      "status": "COMPLETED",
      "objectRid": "ri.ontology.main.object.prj-100"
    },
    {
      "actionId": "act-2024-01-15-d4e5f6",
      "type": "createLink",
      "status": "COMPLETED"
    }
  ]
}
```

## 액션 타입 목록 조회

온톨로지에 정의된 모든 액션 타입 템플릿을 가져와요.

```
GET /api/v2/ontologies/{ontologyApiName}/actionTypes
```

---

## 액션 검증

실제로 실행하지 않고 액션이 성공할지 미리 확인할 수 있어요.

```
POST /api/v2/ontologies/{ontologyApiName}/actions/validate
```

요청 형식은 `apply`와 동일하지만, 실제 변경은 수행되지 않아요.

### 유효한 경우

```json
{ "valid": true, "warnings": [] }
```

### 유효하지 않은 경우

```json
{
  "valid": false,
  "errors": [
    {
      "field": "department",
      "errorCode": "INVALID_VALUE",
      "message": "'Eng' 값은 허용된 값 목록에 없습니다",
      "allowedValues": ["Engineering", "Product", "Marketing", "HR", "Finance"]
    },
    {
      "field": "employeeId",
      "errorCode": "DUPLICATE_PRIMARY_KEY",
      "message": "기본키 'EMP-042'를 가진 객체가 이미 존재합니다"
    }
  ]
}
```

## 액션 템플릿 시스템

액션 템플릿은 동적 표현식 언어를 사용해 재사용 가능한 파라미터화된 액션을 정의해요.

### 템플릿 연산자

| 연산자 | 설명 | 예시 |
|:-------|:-----|:-----|
| `$ref` | 파라미터, 속성, 컨텍스트 값 참조 | `{"$ref": "parameters.fullName"}` |
| `$now` | 현재 타임스탬프 | `{"$now": "ISO8601"}` |
| `$if` | 조건부 표현식 | `{"$if": {"cond": ..., "then": ..., "else": ...}}` |
| `$switch` | 다중 분기 조건 | `{"$switch": {"on": ..., "cases": {...}, "default": ...}}` |
| `$call` | 등록된 함수 호출 | `{"$call": {"fn": "generateId", "args": [...]}}` |

### 템플릿 예시

```json
{
  "apiName": "onboardEmployee",
  "displayName": "직원 온보딩",
  "type": "createObject",
  "objectTypeApiName": "Employee",
  "template": {
    "employeeId": { "$call": { "fn": "generateId", "args": ["EMP"] } },
    "fullName": { "$ref": "parameters.fullName" },
    "department": { "$ref": "parameters.department" },
    "startDate": {
      "$if": {
        "cond": { "$ref": "parameters.startDate" },
        "then": { "$ref": "parameters.startDate" },
        "else": { "$now": "date" }
      }
    },
    "isActive": true,
    "accessLevel": {
      "$switch": {
        "on": { "$ref": "parameters.department" },
        "cases": {
          "Engineering": "full",
          "Product": "full",
          "Marketing": "standard",
          "HR": "restricted"
        },
        "default": "standard"
      }
    }
  }
}
```

## 액션 되돌리기 (Undo)

이전에 실행한 액션을 보상 액션(compensating action)으로 되돌릴 수 있어요.

```
POST /api/v2/ontologies/{ontologyApiName}/actions/{actionId}/undo
```

### 응답

```json
{
  "undoActionId": "act-2024-01-15-undo-a1b2c3",
  "status": "COMPLETED",
  "originalActionId": "act-2024-01-15-a1b2c3"
}
```

:::caution
모든 액션이 되돌리기 가능한 것은 아니에요. `deleteObject` 액션은 이벤트 스토어에 객체의 생성/수정 이벤트가 남아 있을 때만 되돌릴 수 있습니다. 액션 결과의 `undoable` 필드를 꼭 확인해 주세요.
:::

---

## 충돌 해결

동시에 같은 객체를 수정하려 하면, 플랫폼이 **낙관적 잠금(Optimistic Locking)**으로 충돌을 감지해요. 각 객체는 업데이트할 때마다 `version` 번호가 1씩 올라갑니다.

충돌이 감지되면 다음과 같은 에러가 반환돼요.

```json
{
  "errorCode": "CONFLICT",
  "errorName": "ConcurrentModification",
  "parameters": {
    "objectRid": "ri.ontology.main.object.emp-042",
    "expectedVersion": 5,
    "actualVersion": 6
  }
}
```

해결 방법: 객체를 다시 읽어 최신 버전을 가져온 후 액션을 재제출합니다.

## 에러 응답

| 상태 | 에러 코드 | 설명 |
|:-----|:---------|:-----|
| `400` | `InvalidAction` | 액션 페이로드가 잘못됨 |
| `404` | `ObjectNotFound` | 대상 객체가 존재하지 않음 (수정/삭제 시) |
| `409` | `ConcurrentModification` | 다른 액션이 동시에 객체를 수정함 |
| `422` | `ValidationFailed` | 비즈니스 룰 검증 실패 |
