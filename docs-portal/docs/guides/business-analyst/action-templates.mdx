## What This Guide Covers

This guide will walk business analysts through creating and using action templates to automate common data operations, including:

- **Template Concepts** -- What action templates are, when to use them, and how they differ from raw actions
- **Built-in Templates** -- Pre-configured templates for common operations (onboarding, status updates, bulk edits)
- **Template Expressions** -- Using `$ref`, `$now`, `$if`, `$switch`, and `$call` to build dynamic logic
- **Parameter Definitions** -- Defining template parameters with types, defaults, and validation
- **Conditional Logic** -- Branching based on input values or existing object state
- **Template Testing** -- Validating templates before deployment using the validate endpoint
- **Template Versioning** -- Managing template changes without breaking existing workflows
- **Common Patterns** -- Approval workflows, cascading updates, computed fields
