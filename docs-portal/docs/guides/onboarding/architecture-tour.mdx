### Layer Responsibilities

| Layer | Responsibility |
|-------|---------------|

## Request Journey: Reading an Object

Follow a `GET /api/v2/ontologies/acme/objectTypes/Employee/objects/EMP-042` request:

```
1. Client sends HTTP GET
   в”Ӯ
2. BFF Auth Middleware
   в”ңв”Җв”Җ Extract Bearer token
   в”ңв”Җв”Җ Validate against expected tokens
   в””в”Җв”Җ Attach UserPrincipal to request
   в”Ӯ
3. BFF Route Handler (foundry_ontology_v2.py)
   в”ңв”Җв”Җ Parse ontology API name вҶ’ database name
   в”ңв”Җв”Җ Parse object type and primary key
   в””в”Җв”Җ Forward to OMS
   в”Ӯ
4. OMS Route Handler
   в”ңв”Җв”Җ Query Elasticsearch for the object
   в”ңв”Җв”Җ Overlay key specs from PostgreSQL
   в””в”Җв”Җ Return formatted response
   в”Ӯ
5. BFF Response Formatting
   в”ңв”Җв”Җ Convert to Foundry v2 format
   в”ңв”Җв”Җ Add __rid, __primaryKey, __apiName
   в””в”Җв”Җ Return to client
```

**Services touched:** BFF вҶ’ OMS вҶ’ Elasticsearch (+ PostgreSQL for key specs)

**Typical latency:** 10-50ms

## Write Path: Creating an Object

Follow a `POST /api/v2/ontologies/acme/actions/createObject` request:

```
1. Client sends action request
   в”Ӯ
2. BFF Auth + Validation
   в”ңв”Җв”Җ Authenticate user
   в”ңв”Җв”Җ Validate action parameters
   в””в”Җв”Җ Check permissions
   в”Ӯ
3. BFF Action Handler
   в”ңв”Җв”Җ Build action command
   в”ңв”Җв”Җ Generate idempotency key
   в””в”Җв”Җ Submit to action pipeline
   в”Ӯ
4. Action Processing
   в”ңв”Җв”Җ Validate against ontology schema
   в”ңв”Җв”Җ Create immutable event
   в””в”Җв”Җ Persist to event store (S3/MinIO)
   в”Ӯ
5. Event Distribution (Kafka)
   в”ңв”Җв”Җ Publish event to Kafka topic
   в””в”Җв”Җ Multiple consumers receive event
   в”Ӯ
6. Worker Processing (parallel)
   в”Ӯ
   в”ңв”Җв”Җ Instance Worker
   в”Ӯ   в””в”Җв”Җ Write object to PostgreSQL
   в”Ӯ
   в”ңв”Җв”Җ Elasticsearch Indexer
   в”Ӯ   в””в”Җв”Җ Index object for search
   в”Ӯ
   в”ңв”Җв”Җ Projection Worker
   в”Ӯ   в””в”Җв”Җ Update aggregated views
   в”Ӯ
   в””в”Җв”Җ Lineage Worker
       в””в”Җв”Җ Record provenance edges
```

**Services touched:** BFF вҶ’ OMS вҶ’ Kafka вҶ’ (Instance Worker вҶ’ PostgreSQL, ES Indexer вҶ’ Elasticsearch, Projection Worker вҶ’ PostgreSQL)

**Typical latency:** ~100ms for acknowledgment, 1-5s for full consistency across all read models

## Search Path: Finding Objects

Follow a `POST /api/v2/ontologies/acme/objectTypes/Employee/objects/search` request:

```
1. Client sends search query
   в”Ӯ
2. BFF Route Handler
   в”ңв”Җв”Җ Parse SearchJsonQueryV2 query
   в”ңв”Җв”Җ Validate operators and nesting depth (max 3)
   в””в”Җв”Җ Forward to OMS
   в”Ӯ
3. OMS Search Handler
   в”ңв”Җв”Җ Convert to Elasticsearch DSL
   в”ңв”Җв”Җ Apply tenant scoping
   в”ңв”Җв”Җ Execute against Elasticsearch
   в””в”Җв”Җ Format results with __rid, __primaryKey
   в”Ӯ
4. BFF Response
   в”ңв”Җв”Җ Add pagination token
   в””в”Җв”Җ Return to client
```

**Services touched:** BFF вҶ’ OMS вҶ’ Elasticsearch

**Typical latency:** 20-100ms

## Data Import Path

Follow a table import execution:

```
1. Execute import request
   в”Ӯ
2. BFF Connectivity Handler
   в”ңв”Җв”Җ Load connection config
   в”ңв”Җв”Җ Decrypt secrets (AES-GCM)
   в””в”Җв”Җ Select connector adapter
   в”Ӯ
3. Connector Adapter (e.g., PostgreSQL)
   в”ңв”Җв”Җ SQL Query Guard (normalize, inject check)
   в”ңв”Җв”Җ Execute query (300s timeout)
   в””в”Җв”Җ Return result set
   в”Ӯ
4. Sync Mode Processing
   в”ңв”Җв”Җ SNAPSHOT: Replace all data
   в”ңв”Җв”Җ APPEND: Deduplicate and append
   в”ңв”Җв”Җ UPDATE: Upsert by primary key
   в”ңв”Җв”Җ INCREMENTAL: Watermark-based fetch
   в””в”Җв”Җ CDC: Change token tracking
   в”Ӯ
5. Storage
   в”ңв”Җв”Җ Write to lakeFS (versioned commit)
   в”ңв”Җв”Җ Register in Dataset Registry
   в””в”Җв”Җ Publish completion event to Kafka
```

**Services touched:** BFF вҶ’ External Database вҶ’ LakeFS вҶ’ MinIO вҶ’ PostgreSQL вҶ’ Kafka

## Pipeline Execution Path

Follow a pipeline build:

```
1. Pipeline build request
   в”Ӯ
2. Pipeline Worker receives job (from Kafka)
   в”ңв”Җв”Җ Load pipeline definition
   в”ңв”Җв”Җ Normalize nodes and edges
   в””в”Җв”Җ Topological sort (DAG order)
   в”Ӯ
3. Execute nodes in order
   в”Ӯ
   в”ңв”Җв”Җ Input Node
   в”Ӯ   в””в”Җв”Җ Load data from dataset (LakeFS)
   в”Ӯ
   в”ңв”Җв”Җ Transform Node (22 types)
   в”Ӯ   в”ңв”Җв”Җ filter, compute, join, aggregate...
   в”Ӯ   в”ңв”Җв”Җ UDF (sandboxed Python)
   в”Ӯ   в””в”Җв”Җ Schema validation at each step
   в”Ӯ
   в””в”Җв”Җ Output Node
       в”ңв”Җв”Җ Validate schema contract
       в”ңв”Җв”Җ Check expectations (unique, required, FK)
       в””в”Җв”Җ Write to dataset / ontology
   в”Ӯ
4. Post-processing
   в”ңв”Җв”Җ Record build artifacts in S3
   в”ңв”Җв”Җ Update pipeline registry
   в””в”Җв”Җ Publish lineage edges
```

## Service Communication Map

| From | To | Protocol | Purpose |
|------|-----|---------|---------|
| Client | BFF | HTTP | API requests |
| BFF | OMS | HTTP | Schema + search operations |
| BFF | Kafka | Kafka | Event publishing |
| BFF | Redis | Redis | Caching, rate limiting |
| BFF | PostgreSQL | SQL | Configuration, registries |
| BFF | LakeFS | HTTP | Dataset storage |
| Workers | Kafka | Kafka | Event consumption |
| Workers | PostgreSQL | SQL | Read model updates |
| Workers | Elasticsearch | HTTP | Search index updates |
| Workers | MinIO | S3 | File storage |
| OMS | PostgreSQL | SQL | Schema storage |
| OMS | Elasticsearch | HTTP | Search queries |

## Hands-On Exploration

### Verify All Services Are Running

```bash
docker compose ps
```

Look for all services showing `Up (healthy)`.

### Check BFF Health

```bash
curl http://localhost:8080/api/health | jq
```

### Explore the API

```bash
# List ontologies
curl -H "Authorization: Bearer YOUR_TOKEN" \
  http://localhost:8080/api/v2/ontologies | jq

# List object types
curl -H "Authorization: Bearer YOUR_TOKEN" \
  http://localhost:8080/api/v2/ontologies/acme/objectTypes | jq
```

### Monitor Kafka Topics

```bash
docker compose exec kafka kafka-topics \
  --bootstrap-server localhost:29092 --list
```

### Browse MinIO Storage

Open http://localhost:9001 in your browser and log in with MinIO credentials.

### Browse lakeFS

Open http://localhost:48080 in your browser to see data versioning repositories.

### Check Elasticsearch Indices

```bash
curl http://localhost:9200/_cat/indices?v
```

## Key Design Patterns

| Pattern | Where Used | Why |
|---------|-----------|-----|
| **Event Sourcing** | All write operations | Audit trail, undo, rebuild |
| **CQRS** | BFF (write) + OMS (read) | Optimized read/write paths |
| **Outbox Pattern** | Action outbox worker | At-least-once delivery guarantee |
| **Adapter Pattern** | Connector adapters | Pluggable data source support |
| **DAG Execution** | Pipeline executor | Deterministic transform ordering |
| **Sandbox** | UDF runtime | Security isolation for custom code |
