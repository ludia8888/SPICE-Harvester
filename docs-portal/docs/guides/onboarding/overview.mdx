# Onboarding Overview

Welcome to SPICE Harvester! This guide provides a structured onboarding path for new team members. Whether you are a data engineer, platform administrator, integration developer, or business analyst, this page will help you get started quickly.

## What is SPICE Harvester?

SPICE Harvester is an **enterprise ontology and data platform** inspired by Palantir Foundry. It provides:

- **Ontology-driven data model** -- define object types, properties, and relationships in a strongly-typed schema
- **Data ingestion** -- connect to 6 external data sources (PostgreSQL, MySQL, Oracle, Snowflake, SQL Server, Google Sheets) with 5 sync modes
- **Pipeline builder** -- transform data with 22 transform types, UDFs, and 3 execution modes (Preview/Build/Deploy)
- **Foundry v2 API** -- 234+ REST endpoints compatible with Palantir Foundry's API surface
- **Event sourcing** -- every mutation is captured as an immutable event with full audit trail
- **Versioned datasets** -- Git-like branching and versioning for data via LakeFS

## Key Terminology

| Term | Definition |
|------|-----------|
| **Ontology** | Top-level container defining your data model |
| **Object Type** | Schema blueprint (like a database table) |
| **Property** | Typed attribute on an object type |
| **Instance** | Concrete record of an object type |
| **Link Type** | Directed relationship between object types |
| **Action** | Validated, audited write operation |
| **Dataset** | Versioned data object stored in LakeFS |
| **Pipeline** | DAG of transform nodes for data processing |
| **Connection** | Configured link to an external data source |
| **Projection** | Precomputed view optimized for queries |
| **RID** | Resource Identifier (`ri.spice.main.{type}.{id}`) |
| **Event** | Immutable record of a state change |

## Environment Access

### Local Development

```bash
# Clone the repository
git clone <repository-url>
cd SPICE-Harvester/backend

# Start all services
docker compose up -d

# Verify health
curl http://localhost:8080/api/health
```

### API Access

| Service | URL | Purpose |
|---------|-----|---------|
| BFF API | `http://localhost:8080/api` | Primary API gateway |
| OMS API | `http://localhost:8000` | Ontology management |
| MinIO Console | `http://localhost:9001` | Object storage UI |
| lakeFS | `http://localhost:48080` | Data versioning UI |
| Elasticsearch | `http://localhost:9200` | Search engine |

### Getting Credentials

1. Check `.env` or `docker-compose.yml` for default tokens
2. For JWT auth, configure your identity provider (see [Auth Flow](../integration-developer/auth-flow))
3. For development, enable Dev Master Auth (development only)
