## What This Guide Covers

This guide will walk data engineers through configuring ontology schemas in SPICE Harvester, including:

- **Defining Object Types** -- Creating object type definitions with properties, data types, and constraints
- **Configuring Properties** -- Setting up primary keys, required fields, indexed fields, and default values
- **Creating Link Types** -- Establishing relationships between object types with appropriate cardinality
- **Schema Versioning** -- Managing schema evolution without breaking existing data or integrations
- **Validation Rules** -- Defining constraints and validation logic for property values
- **Index Configuration** -- Controlling which properties are indexed in Elasticsearch and how they are analyzed
- **Best Practices** -- Naming conventions, normalization strategies, and performance considerations
