# Class Diagrams

This page presents the key domain model classes in SPICE Harvester. These diagrams reflect the core entities and their relationships as implemented in the ontology management layer.

## Ontology Domain Model

The following class diagram shows the primary entities that make up the ontology schema and instance layers.

```mermaid
classDiagram
    class Ontology {
        +String rid
        +String apiName
        +String displayName
        +String description
        +DateTime createdAt
        +DateTime updatedAt
        +listObjectTypes() ObjectType[]
        +listLinkTypes() LinkType[]
        +getObjectType(apiName) ObjectType
    }

    class ObjectType {
        +String rid
        +String apiName
        +String displayName
        +String pluralDisplayName
        +String primaryKey
        +String ontologyApiName
        +Map~String, Property~ properties
        +DateTime createdAt
        +getProperty(apiName) Property
        +validate(instance) ValidationResult
    }

    class Property {
        +String apiName
        +String displayName
        +DataType dataType
        +Boolean required
        +Boolean indexed
        +Object defaultValue
        +String description
    }

    class DataType {
        +String type
        +DataType itemType
        +Boolean nullable
    }

    class LinkType {
        +String rid
        +String apiName
        +String displayName
        +String sourceObjectType
        +String targetObjectType
        +Cardinality cardinality
        +String ontologyApiName
    }

    class Instance {
        +String rid
        +String primaryKey
        +String objectTypeApiName
        +String ontologyApiName
        +Map~String, Any~ properties
        +DateTime createdAt
        +DateTime updatedAt
        +Long version
        +getProperty(name) Any
        +toFoundryV2Format() Object
    }

    Ontology "1" --> "*" ObjectType : contains
    Ontology "1" --> "*" LinkType : contains
    ObjectType "1" --> "*" Property : defines
    Property "1" --> "1" DataType : has type
    LinkType "1" --> "1" ObjectType : source
    LinkType "1" --> "1" ObjectType : target
    ObjectType "1" --> "*" Instance : instantiates
```

## Action System Model

Actions represent all write operations in the platform. They are validated, executed, and recorded as events.

```mermaid

        +String actionId
        +ActionType type
        +String ontologyApiName
        +String objectTypeApiName
        +Map~String, Any~ parameters
        +String userId
        +DateTime submittedAt
        +ActionStatus status
        +validate() ValidationResult
        +execute() ActionResult
        +undo() Action
    }

    class ActionType {
        <<enumeration>>
        CREATE_OBJECT
        EDIT_OBJECT
        DELETE_OBJECT
        CREATE_LINK
        DELETE_LINK
        BULK_ACTION
    }

    class ActionTemplate {
        +String templateId
        +String apiName
        +String displayName
        +ActionType type
        +Map~String, ParameterDef~ parameters
        +Expression[] rules
        +Expression[] sideEffects
        +evaluate(context) Action
    }

    class ActionResult {
        +String actionId
        +Boolean success
        +String objectRid
        +Map~String, Any~ modifiedProperties
        +Event[] emittedEvents
        +String errorCode
        +String errorMessage
    }

    class ValidationResult {
        +Boolean valid
        +ValidationError[] errors
        +ValidationWarning[] warnings
    }

    class ActionStatus {
        <<enumeration>>
        PENDING
        VALIDATING
        EXECUTING
        COMPLETED
        FAILED

    }

    Action --> ActionType : has type
    Action --> ActionResult : produces
    Action --> ActionStatus : has status
    Action --> ValidationResult : validated by
    ActionTemplate --> Action : generates
    ActionTemplate --> ActionType : defines type
```

## Relationship Summary

The domain model follows these key relationships:

- An **Ontology** contains multiple **ObjectTypes** and **LinkTypes**.
- Each **ObjectType** defines multiple **Properties**, each with a **DataType**.
- **Instances** are concrete records conforming to an ObjectType schema.
- **Actions** mutate instances and produce **Events**.
- **Events** are stored in the **EventStore** and consumed by **EventConsumers** (workers).
- **Pipelines** read from **Sources**, apply **Transforms**, and write to **Sinks** (which submit actions).
- **Lineage** records provenance relationships between **LineageNodes** via typed **LineageEdges**.
