Installation

This guide covers a complete SPICE Harvester installation, including system requirements, service configuration, database setup, and production hardening.

System Requirements​

Hardware​

EnvironmentCPURAMDiskNetwork
Development4 cores16 GB20 GB SSDlocalhost only
Staging8 cores32 GB100 GB SSDPrivate network
Production16+ cores64+ GB500+ GB NVMeLoad-balanced

Software​

ComponentRequired VersionNotes
Docker Engine24.0+Docker Desktop or standalone
Docker Compose2.20+V2 plugin format
Python3.11+For CLI tools and scripts
Node.js20+ LTSFor BFF and frontend tooling
PostgreSQL client15+Optional, for direct DB access

Operating Systems​

  • macOS: 13 Ventura or later (Apple Silicon and Intel)
  • Linux: Ubuntu 22.04+, Debian 12+, RHEL 9+, Amazon Linux 2023
  • Windows: Windows 11 with WSL2 (native Windows is not supported)

Docker Compose Setup​

Full Stack​

The default Docker Compose configuration starts all 33+ services:

docker compose up -d

Service Profiles​

Use Docker Compose profiles to start subsets of services:

# Core only: API servers + data stores (no observability)
docker compose --profile core up -d

# Core + observability
docker compose --profile core --profile observability up -d

# Core + workers (for event processing)
docker compose --profile core --profile workers up -d

Available profiles:

ProfileServicesPorts
coreOMS, BFF, Funnel, Admin, PostgreSQL, Elasticsearch, Redis, Kafka, MinIO8000-8003, 5432, 9200, 6379, 9092, 9000
workersEvent Processor, Projection Builder, Pipeline Runner, Lineage Worker, and 9 moreInternal only
observabilityGrafana, Prometheus, Jaeger, OTEL Collector, AlertManager, Loki13000, 19090, 16686

Environment Variables​

Core Configuration​

Create a .env file in the project root. The following variables control essential behavior:

# ─── Service Ports ───────────────────────────────────────
OMS_PORT=8000
BFF_PORT=8002
FUNNEL_PORT=8003
ADMIN_PORT=8004

# ─── PostgreSQL ──────────────────────────────────────────
POSTGRES_HOST=postgres
POSTGRES_PORT=5432
POSTGRES_DB=spice_harvester
POSTGRES_USER=spice
POSTGRES_PASSWORD=changeme_in_production

# ─── Elasticsearch ───────────────────────────────────────
ELASTICSEARCH_HOST=elasticsearch
ELASTICSEARCH_PORT=9200
ELASTICSEARCH_INDEX_PREFIX=spice_

# ─── Redis ───────────────────────────────────────────────
REDIS_HOST=redis
REDIS_PORT=6379
REDIS_PASSWORD=
REDIS_DB=0

# ─── Kafka ───────────────────────────────────────────────
KAFKA_BOOTSTRAP_SERVERS=kafka:9092
KAFKA_CONSUMER_GROUP=spice-harvester
KAFKA_AUTO_OFFSET_RESET=earliest

# ─── MinIO (S3-compatible Event Store) ───────────────────
MINIO_ENDPOINT=minio:9000
MINIO_ACCESS_KEY=minioadmin
MINIO_SECRET_KEY=minioadmin
MINIO_BUCKET=spice-events

# ─── Authentication ─────────────────────────────────────
AUTH_SECRET_KEY=generate-a-strong-secret-here
AUTH_TOKEN_EXPIRY=3600
AUTH_ISSUER=spice-harvester

# ─── Logging ─────────────────────────────────────────────
LOG_LEVEL=INFO
LOG_FORMAT=json
LOG_OUTPUT=stdout

Production Overrides​

For production deployments, override the following:

# Strong secrets (generate with: openssl rand -hex 32)
POSTGRES_PASSWORD=<generated-secret>
AUTH_SECRET_KEY=<generated-secret>
MINIO_SECRET_KEY=<generated-secret>

# External URLs
BFF_PUBLIC_URL=https://api.your-domain.com
CORS_ORIGINS=https://app.your-domain.com

# Performance tuning
OMS_WORKERS=4
BFF_WORKERS=4
FUNNEL_WORKERS=2
POSTGRES_MAX_CONNECTIONS=100
ELASTICSEARCH_SHARDS=3
ELASTICSEARCH_REPLICAS=1
KAFKA_PARTITIONS=12

Database Migrations​

Initial Setup​

Run Alembic migrations to create the database schema:

docker compose exec oms python -m alembic upgrade head

Seed Data​

Load demo data for development and testing:

# Full demo dataset with sample ontology
docker compose exec oms python -m spice.cli seed --demo

# Minimal seed (ontology schema only, no instances)
docker compose exec oms python -m spice.cli seed --minimal

Migration Management​

# Check current migration version
docker compose exec oms python -m alembic current

# View migration history
docker compose exec oms python -m alembic history

# Create a new migration
docker compose exec oms python -m alembic revision --autogenerate -m "description"

# Downgrade one step
docker compose exec oms python -m alembic downgrade -1

Elasticsearch Index Setup​

Elasticsearch indices are created automatically on first boot. To manually recreate them:

docker compose exec oms python -m spice.cli reindex --all

Health Checks​

Every service exposes a /health endpoint. Use the CLI to verify all services:

docker compose exec oms python -m spice.cli health --all

Common Issues​

Port Conflicts​

If a port is already in use, override it in your .env file:

# Example: move BFF to port 8082
BFF_PORT=8082

Then restart the affected service:

docker compose up -d bff

Elasticsearch Startup Failure​

Elasticsearch requires vm.max_map_count to be at least 262144 on Linux:

# Check current value
sysctl vm.max_map_count

# Set temporarily
sudo sysctl -w vm.max_map_count=262144

# Set permanently (add to /etc/sysctl.conf)
echo "vm.max_map_count=262144" | sudo tee -a /etc/sysctl.conf

On macOS with Docker Desktop, this is handled automatically.

Kafka Connection Issues​

If workers cannot connect to Kafka, verify the advertised listeners:

docker compose logs kafka | grep "advertised.listeners"

Ensure KAFKA_ADVERTISED_LISTENERS matches your network configuration.

Database Migration Failures​

If migrations fail, check the current state and resolve conflicts:

# Check for pending migrations
docker compose exec oms python -m alembic current

# Stamp the database to a known state (use with caution)
docker compose exec oms python -m alembic stamp head

Memory Pressure​

If services are being OOM-killed, adjust Docker resource limits:

# docker-compose.override.yml
services:
  elasticsearch:
    deploy:
      resources:
        limits:
          memory: 4g
  postgres:
    deploy:
      resources:
        limits:
          memory: 2g

Resetting Everything​

To completely reset the platform and start fresh:

docker compose down -v    # Stop services and remove volumes
docker compose up -d      # Start fresh
docker compose exec oms python -m alembic upgrade head
docker compose exec oms python -m spice.cli seed --demo
caution

The -v flag deletes all persistent data including database contents, Elasticsearch indices, and Kafka topics. Use only in development.
