Foundry v2 Search API

The Search API allows you to query objects using the SearchJsonQueryV2 format, which supports 13 operators for building expressive filters. Search is powered by Elasticsearch and returns results in the standard Foundry v2 object format.

Search Objects​

POST /api/v2/ontologies/{ontologyApiName}/objects/{objectTypeApiName}/search

Path Parameters​

ParameterTypeRequiredDescription
ontologyApiNamestringYesThe API name of the ontology
objectTypeApiNamestringYesThe API name of the object type to search

Request Body​

{
  "where": { ... },
  "orderBy": [
    { "field": "startDate", "direction": "desc" }
  ],
  "select": ["fullName", "department", "startDate"],
  "pageSize": 25,
  "pageToken": null
}
FieldTypeRequiredDescription
whereSearchJsonQueryV2YesFilter expression using search operators
orderByarrayNoSort order. Each element has field and direction (asc or desc)
selectstring[]NoProperties to include in results
pageSizeintegerNoResults per page (default: 100, max: 1000)
pageTokenstringNoOpaque token for the next page

SearchJsonQueryV2 Operators​

The where clause uses a recursive JSON structure. Each operator is identified by a type field.

Comparison Operators​

eq -- Equals​

Match objects where a field equals a specific value.

{
  "type": "eq",
  "field": "department",
  "value": "Engineering"
}

gt -- Greater Than​

{
  "type": "gt",
  "field": "salary",
  "value": 100000
}

gte -- Greater Than or Equal​

{
  "type": "gte",
  "field": "startDate",
  "value": "2023-01-01"
}

lt -- Less Than​

{
  "type": "lt",
  "field": "salary",
  "value": 200000
}

lte -- Less Than or Equal​

{
  "type": "lte",
  "field": "startDate",
  "value": "2024-12-31"
}

isNull -- Null Check​

Match objects where a field is null (or not null).

{
  "type": "isNull",
  "field": "department",
  "value": true
}

Set value to false to match objects where the field is not null.

Text Search Operators​

containsAnyTerm -- Any Term Match​

Match objects where a text field contains any of the specified terms. Terms are analyzed (lowercased, stemmed) before matching.

{
  "type": "containsAnyTerm",
  "field": "fullName",
  "value": "jane john"
}

This matches objects where fullName contains "jane" OR "john".

containsAllTerms -- All Terms Match​

Match objects where a text field contains all of the specified terms, in any order.

{
  "type": "containsAllTerms",
  "field": "description",
  "value": "data pipeline"
}

This matches objects where description contains both "data" AND "pipeline" (in any order).

containsAllTermsInOrder -- Phrase Match​

Match objects where a text field contains all terms in the exact order specified.

{
  "type": "containsAllTermsInOrder",
  "field": "description",
  "value": "data transformation pipeline"
}

containsAllTermsInOrderPrefixLastTerm -- Prefix Phrase Match​

Like containsAllTermsInOrder, but the last term is treated as a prefix. Useful for autocomplete/typeahead functionality.

{
  "type": "containsAllTermsInOrderPrefixLastTerm",
  "field": "fullName",
  "value": "jane d"
}

This matches "Jane Doe", "Jane Davis", etc.

startsWith -- Prefix Match​

Match objects where a field value starts with a specific prefix. Works on keyword (non-analyzed) fields.

{
  "type": "startsWith",
  "field": "employeeId",
  "value": "EMP-04"
}

Logical Operators​

and -- Logical AND​

Combine multiple conditions. All must match.

{
  "type": "and",
  "value": [
    { "type": "eq", "field": "department", "value": "Engineering" },
    { "type": "gte", "field": "salary", "value": 120000 },
    { "type": "eq", "field": "isActive", "value": true }
  ]
}

or -- Logical OR​

Combine multiple conditions. At least one must match.

{
  "type": "or",
  "value": [
    { "type": "eq", "field": "department", "value": "Engineering" },
    { "type": "eq", "field": "department", "value": "Product" }
  ]
}

not -- Logical NOT​

Negate a condition.

{
  "type": "not",
  "value": {
    "type": "eq",
    "field": "department",
    "value": "HR"
  }
}

Complete Request/Response Example​

Request​

Find active engineers hired after 2023 with salary above 100k, sorted by start date descending:

curl -X POST \
  "http://localhost:8002/api/v2/ontologies/acme/objects/Employee/search" \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "where": {
      "type": "and",
      "value": [
        {
          "type": "eq",
          "field": "department",
          "value": "Engineering"
        },
        {
          "type": "gte",
          "field": "startDate",
          "value": "2023-01-01"
        },
        {
          "type": "gt",
          "field": "salary",
          "value": 100000
        },
        {
          "type": "eq",
          "field": "isActive",
          "value": true
        }
      ]
    },
    "orderBy": [
      { "field": "startDate", "direction": "desc" }
    ],
    "select": ["fullName", "department", "startDate", "salary"],
    "pageSize": 10
  }'

Response​

{
  "data": [

      "__rid": "ri.ontology.main.object.emp-100",

      "__rid": "ri.ontology.main.object.emp-042",

Operator Reference Table​

OperatorTypeFieldsDescription
eqComparisonAll typesExact equality match
gtComparisonNumeric, Date, TimestampGreater than
gteComparisonNumeric, Date, TimestampGreater than or equal
ltComparisonNumeric, Date, TimestampLess than
lteComparisonNumeric, Date, TimestampLess than or equal
isNullComparisonAll typesNull/not-null check
containsAnyTermTextString (analyzed)Any term matches
containsAllTermsTextString (analyzed)All terms match (any order)
containsAllTermsInOrderTextString (analyzed)Phrase match
containsAllTermsInOrderPrefixLastTermTextString (analyzed)Prefix phrase match
startsWithTextString (keyword)Prefix match on keyword field
andLogicalN/AAll conditions must match
orLogicalN/AAt least one condition must match
notLogicalN/ANegate a condition
note

The not operator wraps a single query, while and and or wrap an array of queries. Logical operators can be nested to arbitrary depth.

Error Responses​

StatusError CodeDescription
400InvalidQueryThe search query is malformed or uses an unsupported operator
400InvalidFieldThe specified field does not exist on the object type
400InvalidValueThe value type does not match the field type
404ObjectTypeNotFoundThe object type does not exist
404OntologyNotFoundThe ontology does not exist
