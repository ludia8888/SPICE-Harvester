Error Codes Reference

SPICE Harvester defines 290+ structured error codes across all subsystems. This page documents the error format, naming conventions, severity levels, retry policies, and common error codes.

Error Response Format​

All API errors return a structured JSON response:

{
  "errorCode": "NOT_FOUND",
  "errorName": "ObjectNotFound",
  "errorInstanceId": "err-2024-01-15-a1b2c3d4",
  "parameters": {
    "objectType": "Employee",
    "primaryKey": "EMP-999"
  }
}
FieldTypeDescription
errorCodestringMachine-readable code following the taxonomy structure
errorNamestringHuman-readable error name in PascalCase
errorInstanceIdstringUnique identifier for this error occurrence. Use this when contacting support.
parametersobjectContext-specific details for debugging. Contents vary by error type.

Error Code Structure​

Error codes follow the pattern:

SHV-{SUBSYSTEM}-{DOMAIN}-{CLASS}-{NUMBER}
SegmentDescriptionExamples
SHVPlatform prefix (SPICE Harvester)Always SHV
SUBSYSTEMService subsystemONT, ACT, SRH, PIP, LIN, AUTH, SYS
DOMAINFunctional domainOBJ, SCH, QRY, IDX, EVT, CFG
CLASSError classificationVAL, NFD, CON, PRM, INT, TMO
NUMBERSequential number001-999

Example: SHV-ONT-OBJ-NFD-001 = SPICE Harvester, Ontology subsystem, Object domain, Not Found class, error #001.

Retry Policies​

General Retry Strategy​

For retryable errors, use exponential backoff with jitter:

delay = min(base_delay * 2^attempt + random_jitter, max_delay)
ParameterDefaultDescription
base_delay100 msInitial retry delay
max_delay30 sMaximum delay between retries
max_attempts3Maximum number of retry attempts
jitter0-100 msRandom jitter to prevent thundering herd

Handling Errors in Client Code​

Python Example​

import requests
import time

def call_api_with_retry(url, headers, max_retries=3):
    for attempt in range(max_retries):
        response = requests.get(url, headers=headers)

        if response.status_code == 200:
            return response.json()

        error = response.json()
        error_class = error["errorCode"].split("-")[4] if "-" in error["errorCode"] else ""

        # Non-retryable errors
        if error_class in ("VAL", "NFD", "PRM", "DUP"):
            raise ApiError(error)

        # Rate limited - respect Retry-After header
        if response.status_code == 429:
            wait = int(response.headers.get("Retry-After", 30))
            time.sleep(wait)
            continue

        # Retryable errors - exponential backoff
        delay = min(0.1 * (2 ** attempt), 30)
        time.sleep(delay)

    raise ApiError({"errorCode": "MAX_RETRIES", "errorName": "MaxRetriesExceeded"})
