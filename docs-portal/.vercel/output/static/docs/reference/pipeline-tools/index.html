Pipeline Tools Reference

This page documents the pipeline builder's transform types, MCP tools, and agent-allowed tools for automating data workflows.

Pipeline Transform Types​

The pipeline builder supports 22+ transform types organized into four categories. Each transform receives a data frame as input and produces a data frame as output.

Structural Transforms​

These transforms reshape the data without altering individual values.

TransformDescriptionKey Parameters
mapApply a mapping function to each rowexpression: mapping expression
filterRemove rows that do not match a predicatepredicate: filter expression
flattenExpand nested arrays into individual rowsfield: array field to flatten
groupByGroup rows by one or more keyskeys: grouping fields, aggregations: aggregate expressions
sortOrder rows by one or more fieldsfields: sort fields, order: asc or desc
distinctRemove duplicate rowsfields: fields to check for uniqueness
limitTake the first N rowscount: maximum number of rows
skipSkip the first N rowscount: number of rows to skip
unionCombine rows from multiple sourcessources: array of source references
pivotPivot rows into columnsrowKey, columnKey, valueField

Enrichment Transforms​

These transforms add or modify data using external sources.

TransformDescriptionKey Parameters
lookupEnrich rows by looking up values from an object typeobjectType, lookupKey, returnFields
joinJoin two data frames on matching keysright: second data source, joinKey, joinType
geocodeConvert addresses to geographic coordinatesaddressField, provider
resolveResolve entity references to canonical recordsentityField, strategy
computeAdd computed columns using expressionscolumns: map of column name to expression

Validation Transforms​

These transforms ensure data quality.

TransformDescriptionKey Parameters
validateCheck rows against schema or custom rulesrules: validation rule set
deduplicateRemove or merge duplicate recordsmatchFields, strategy (keep_first, keep_last, merge)
normalizeStandardize field valuesfield, normalization: (trim, lowercase, titlecase, strip_html)
typecastConvert field data typesconversions: map of field to target type

Output Transforms​

These transforms write results to the ontology.

TransformDescriptionKey Parameters
upsertObjectsCreate or update object instancesobjectType, primaryKeyField, propertyMapping
createLinksCreate link instances between objectslinkType, sourceKeyField, targetKeyField
publishEventPublish custom events to Kafkatopic, eventType, payloadMapping

Pipeline Definition Format​

Pipelines are defined as JSON or YAML:

{
  "name": "employee-import",
  "description": "Import employee records from CSV",
  "source": {
    "type": "file",
    "format": "csv",
    "path": "s3://imports/employees.csv",
    "options": {
      "header": true,
      "delimiter": ","
    }
  },
  "transforms": [
    {
      "type": "typecast",
      "config": {
        "conversions": {
          "salary": "double",
          "startDate": "date",
          "isActive": "boolean"
        }
      }
    },
    {
      "type": "normalize",
      "config": {
        "field": "fullName",
        "normalization": "titlecase"
      }
    },
    {
      "type": "validate",
      "config": {
        "rules": [
          { "field": "employeeId", "rule": "required" },
          { "field": "employeeId", "rule": "pattern", "pattern": "^EMP-\\d{3,}$" },
          { "field": "salary", "rule": "range", "min": 0, "max": 1000000 }
        ]
      }
    },
    {
      "type": "deduplicate",
      "config": {
        "matchFields": ["employeeId"],
        "strategy": "keep_last"
      }
    },
    {
      "type": "upsertObjects",
      "config": {
        "objectType": "Employee",
        "primaryKeyField": "employeeId",
        "propertyMapping": {
          "employeeId": "employeeId",
          "fullName": "fullName",
          "department": "department",
          "startDate": "startDate",
          "salary": "salary",
          "isActive": "isActive"
        }
      }
    }
  ]
}

MCP Tools​

The Model Context Protocol (MCP) tools provide an interface for AI agents to interact with the pipeline system.

ToolDescriptionParameters
pipeline.createCreate a new pipeline definitionname, source, transforms
pipeline.runExecute a pipelinepipelineId, dryRun (optional)
pipeline.statusCheck pipeline execution statusexecutionId
pipeline.listList all pipeline definitionspageSize, pageToken
pipeline.getGet a pipeline definitionpipelineId
pipeline.deleteDelete a pipeline definitionpipelineId
pipeline.validateValidate a pipeline definition without executingpipeline (definition object)
pipeline.previewExecute a pipeline on a sample of datapipelineId, sampleSize
ontology.listObjectTypesList object types for mappingontologyApiName
ontology.getObjectTypeGet object type schemaontologyApiName, objectTypeApiName
search.querySearch objects for enrichment lookupsontologyApiName, objectType, where

Agent Allowed Tools​

When running pipelines in agent mode (via AI-assisted pipeline building), the following tools are available to the agent:

Tool CategoryToolsDescription
Readontology.list, ontology.get, objectType.list, objectType.getRead ontology metadata
Searchsearch.query, search.aggregateQuery existing data
Previewpipeline.preview, pipeline.validateTest pipeline definitions
Executepipeline.run (with approval)Run pipelines (requires user confirmation)
Monitorpipeline.status, pipeline.logsMonitor running pipelines
note

The agent cannot directly modify ontology schemas, delete objects, or perform administrative actions. These require explicit user interaction through the Admin service.
